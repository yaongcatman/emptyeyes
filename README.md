### 1. Python 기초 
* **변수(Variable)**: 내가 나중에 다시 쓰고 싶은 소중한 정보를 담아두는 '이름표 붙은 상자'입니다.
* **자료형(Data Types)**: 상자에 무엇을 넣을지 결정하는 규칙입니다.
    * `int`: 숫자 구슬 (1, 2, 3...)
    * `str`: 글자 카드 ("안녕", "원숭이")
    * `bool`: 예/아니오 스티커 (참/거짓)
    * `list`: 여러 물건을 한 줄로 세운 '기차'
    * `dict`: 단어와 뜻이 짝궁인 '낱말 사전'
* **조건문(if/elif/else)**: 갈림길에서 어디로 갈지 정하는 '표지판'입니다.
* **정의**: 조건문은 컴퓨터가 상황에 따라 어떤 행동을 할지 스스로 판단하게 만드는 '마법의 표지판'입니다.
* **활용 방법**:
    * **if (만약에)**: "만약 점수가 100점이라면?"처럼 가장 먼저 확인하는 조건입니다.
    * **elif (그게 아니라 만약에)**: 첫 번째 조건이 아닐 때, "그럼 혹시 80점 이상이니?"라고 다시 한번 물어보는 두 번째 조건입니다.
    * **else (나머지 전부)**: 위에서 말한 조건들에 하나도 해당하지 않을 때 "그럼 그냥 이렇게 해!"라고 마지막으로 정해주는 약속입니다.
* **사용 이유**: 이 표지판이 있어야 프로그램이 똑똑하게 상황에 맞춰 다른 동작(예: 정답이면 폭죽, 오답이면 위로 멘트)을 수행할 수 있습니다.
```bash
banana_count = 5  # 내가 가진 바나나 개수

if banana_count >= 10:
    print("🍌 원숭이가 너무 행복해서 춤을 춰요! (우끼끼!)")
elif banana_count >= 5:
    print("🐵 원숭이가 기분이 좋아요! (우끼!)")
else:
    print("😭 원숭이가 배가 고파서 울고 있어요... (우우..)")
```

* **반복문(Loops)**: 
    * `for`: "10번만 돌아!"라고 정해진 만큼만 타는 '회전목마'입니다.
    * `while`: "전기가 켜져 있는 동안 계속 돌아!"라고 하는 '선풍기'입니다.
* **함수(Function)**: 재료를 넣으면 맛있는 결과물이 나오는 '자동 요리 기계'입니다.

* **개념**: 함수는 특정한 일을 대신 해주는 '마법 상자'입니다.
* **설명**:
    * **매개변수(입력)**: 상자 속으로 쏙 집어넣는 **'동전'**입니다.

# 숫자에 10을 더해주는 '더하기 상자' 함수입니다.
def plus_ten(number):      # 'number'라는 숫자를 재료로 받아요.

result = plus_ten(5)       # 상자에 '5'를 넣었어요.

### 💡 클래스와 객체
* **클래스(Class)**: 똑같은 특징을 가진 물건들을 계속 찍어내기 위해 만든 **'공통 설계도'**입니다. 설계도가 있으면 수백 개의 로봇도 똑같이 만들 수 있습니다.
* **__init__ 메서드**: 로봇이 공장에서 처음 만들어질 때, 이름이나 색깔을 정해주는 **'초기 설정'** 단계입니다. * **self**: "이 로봇은 내 거야!"라고 말하는 것처럼, 여러 로봇 중 **'지금 이 로봇'** 자신을 가리키는 약속입니다.
* **속성(Attribute)과 메서드(Method)**:
    * **속성**: 로봇의 이름, 키, 색깔 같은 **'상태'** (특징)
    * **메서드**: 로봇이 하는 발사, 걷기, 춤추기 같은 **'동작'** (기능)

* **초간단 예시 코드**:
```bash
# '로봇'을 만드는 설계도(클래스)입니다.
class Robot:
    def __init__(self, name):     # [초기화] 로봇이 태어날 때 이름을 지어줘요.
        self.name = name          # [속성] 이 로봇의 이름은 입력받은 이름이 돼요.

    def say_hello(self):          # [메서드] 로봇이 인사하는 기능을 만들어요.
        print(f"치이익... 안녕! 나는 {self.name} 로봇이야.")

# 설계도로 실제 로봇(객체)을 만들어요.
my_robot = Robot("공들이")         # '공들이'라는 이름의 로봇 탄생!
my_robot.say_hello()              # 로봇이 인사를 합니다.
```

### 3. 파일 입출력 및 예외 처리 (기록과 안전제일)
* **파일 입출력**: 오늘 있었던 점수를 일기장(파일)에 적어두고, 내일 다시 읽어보는 과정입니다.
* **JSON**: 친구와 일기를 바꿔 볼 때, 서로 알아볼 수 있게 약속한 '비밀 글자 모양'입니다.
* **예외 처리(try/except)**: 실수로 엉뚱한 버튼을 눌러도 컴퓨터가 "깜짝이야!" 하고 놀라지 않게 미리 준비하는 '안전 그물'입니다.

### 4. Git 기초 (타임머신 타기)
* **Git의 필요성**: 실수로 코드를 지워도 다시 과거로 돌아갈 수 있게 도와주는 든든한 친구입니다.
* **주요 명령어**:
    * `init`: 이제부터 사진을 찍을 준비를 합니다.
    * `add`: 사진 찍을 물건을 책상 위에 올립니다. (장바구니)
    * `commit`: "찰칵!" 사진을 찍어 기록을 남깁니다.
    * `push`: 찍은 사진을 친구들도 볼 수 있게 인터넷 앨범(GitHub)에 올립니다.
    * `pull`: 친구가 올린 새로운 사진을 내 앨범으로 가져옵니다.
    * `clone`: 친구의 앨범을 통째로 복사해 옵니다.
* **브랜치와 병합**: 원래 길에서 잠시 옆길로 가서 새로운 놀이터를 만들고(Branch), 다 만들면 원래 길과 연결하는(Merge) 마법입니다.

# 🎯 나만의 퀴즈 게임: 자아성찰 열고개 (ukiki_quiz)

## 1. 프로젝트 개요
본 프로젝트는 Python 입문 과정에서 학습한 기초 문법, 객체 지향 프로그래밍(Class), 파일 입출력(JSON), 그리고 Git의 버전 관리 기능을 실습하기 위해 제작된 터미널 기반 퀴즈 프로그램입니다.

## 2. 퀴즈 주제와 선정 이유
- **주제**: "나 자신"을 찾아가는 자아성찰 열고개
- **선정 이유**: 프로그래밍 기술을 배우는 과정 또한 하나의 자아성찰 과정이라고 생각했습니다. 사용자가 10단계의 힌트를 통해 정답인 '나 자신' 혹은 '나'를 맞춰가는 과정을 통해, 스스로의 소중함을 다시 한번 느끼게 하고자 이 주제를 선정하게 되었습니다.

### 3. 실행 방법
프로젝트 루트 디렉토리에서 아래 명령어를 입력하여 실행합니다.
```bash
python3 main.py
```
## 4. 기능 목록
메뉴 시스템: 1~5번 번호 입력을 통해 퀴즈 풀기, 추가, 목록 확인, 점수 확인 기능을 제공합니다.

퀴즈 풀기: 저장된 자아성찰 퀴즈를 순차적으로 풀며, 정답 여부를 즉시 확인하고 최종 결과를 출력합니다.

퀴즈 추가: 사용자가 직접 새로운 문제, 선택지(4개), 정답 번호를 입력하여 데이터에 반영할 수 있습니다.

퀴즈 목록: 현재 시스템에 저장된 전체 퀴즈 리스트를 번호와 함께 출력합니다.

점수 기록 및 저장: 최고 점수(최대 정답 수)를 기록하며, 프로그램을 종료해도 데이터가 유지됩니다.

예외 처리: 숫자 변환 실패, 잘못된 범위 입력, 빈 입력 처리 및 Ctrl+C 등에 의한 비정상 종료 시 안전 저장 로직을 포함합니다.
